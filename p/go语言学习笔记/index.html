初始化mod文件

mod文件可以在我们的项目处在GitHub的时候快捷的被他人引用。通过一下命令可以快速添加初始化应该mod文件。

1
2

$ go mod init example/hello
go: creating new go.mod: module example/hello

其中的example/hello是项目地址，如果有GitHub可以是github.com/你的账号名称/此项目对应的仓库

hello world

在刚刚创建完成的go项目中创建应该以.go结尾的go代码文件。

下面我会使用hello.go

这是应该古老的仪式，据说起源于一本漫画书中有一个人说了一句“Hello world!”。so，以下代码是go的输出hello world。

1
2
3
4
5
6
7

package main // 声明一个包（包是一种用于对 函数，它由同一目录中的所有文件组成）。main

import "fmt" // 导入fmt包，这个包在go中用于处理一些常见需求

func main() { // 主函数main
    fmt.Println("Hello, World!") // 输出Hello, World!
}

将上面的代码复制到hello.go并保存。

构建exe

还是以上面的hello.go打比方

1.构建单个exe：

使用go build <go代码文件名>

这样构建的exe是以文件名命名的。

2.构建整个项目（推荐）

使用go build .

这样构建的exe是以项目名命名的。

变量与常量

关键字

变量：使用var关键字声明 常量：使用const关键字声明

语法

// 变量声明
var 变量名称 变量类型 = 变量值 // 最常见的声明方式
var 变量名称 := 变量值 // 自动推断类型
var 变量名称,变量名称,··· := 变量值,变量值,··· // 同时声明多个变量

// 常量声明
const 常量名称 常量类型 = 常量值 // 最常见的声明方式
const 常量名称 := 常量值 // 自动推断类型
const 常量名称,常量名称,··· := 常量值,常量值,··· // 同时声明多个常量
